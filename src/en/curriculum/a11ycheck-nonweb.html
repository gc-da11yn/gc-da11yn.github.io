--- title: Accessibility Compliance Assessments checklist (non-web / software) layout: layouts/base.njk description: Accessibility resources for non-Web or software ---

1 – Keyboard related items

  • 1.1 – Are there keyboard shortcuts available for all major functions of the software?
  • 1.2 – Is the Tab-Order in a logical left to right, top to bottom order?
  • 1.3 – Can all the controls be reached with the Tab key or other navigation keys without using a mouse (Ex: F6 for task panes)?
  • 1.4 – Is there a well-defined visual focus indicator and insertion point that moves with standard keyboard navigation controls?
  • 1.5 – Is there a keyboard navigation indicator (“_” - underlined access key) for every menu item and every command control?
  • 1.6 – Is there a menu equivalent for every function found in all toolbars and format bars?

2 – Colours related items

  • 2.1 Does the software use colour as an enhancement, not as the only way to convey information or indicate an action?

3 – Cognitive related items

  • 3.1 – Does the software avoid the use of acronyms that are not defined elsewhere?

4 – Outputs and Error Messages related items

  • 4.1 – Can the documents be generated in more than one format?

5 – Screen Texts related items

  • 5.1 – Does every window, object, and control have a clearly named label?
  • 5.2 – Do all toolbar controls have contextual help or ToolTips?
  • 5.3 – Does each dialog have a unique and significant title?

6 – Icons, Images and Cursor related items

  • 6.1 – Does the software use the standard O/S icons for common functions (Ex: The diskette icon for Save)?
  • 6.2 – Does the software use the same icons throughout the software for the same functions?
  • 6.3 – Does the software avoid use of patterned backgrounds?
  • 6.4 – When images can be added, does the software allow a description of the image to be included?
  • 6.5 – Does the software use the O/S defined mouse cursors?

7 – Accessibility options related items

  • 7.1 – Do the O/S accessibility features continue working when the software is running?

8 – Multimedia, sounds and animations related items

  • 8.1 – Does the software have a visual or textual alternative for information that is available in audio format?
  • 8.2 – Can the volume be changed or muted?
  • 8.3 – Does the software have textual description for video format?
  • 8.4 – Does the software allow animations to be disabled?

9 – Documentation related items

  • 9.1 – Does the documentation clearly state all keyboard shortcuts that can be used with the software?
  • 9.2 – Does the documentation provide clear and precise step-by-step keyboard instructions for all the functions of the program?
  • 9.3 – Does the documentation include text to describe images and videos?

10 – Timing related items

  • 10.1 – Does the software avoid the use of blinking text, objects or other similar elements?
